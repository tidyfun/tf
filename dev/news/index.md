# Changelog

## tf (development version)

## tf 0.5.0

CRAN release: 2026-07-14

### Internal

- [`tf_rgp()`](https://tidyfun.github.io/tf/dev/reference/tf_rgp.md) no
  longer depends on `mvtnorm` or `pracma`. GP draws are now generated by
  a direct eigen-based sampler (the symmetric square root
  `V D^{1/2} V^T` of the covariance) that reproduces the previous
  `mvtnorm::rmvnorm(method = "eigen")` draws bit-for-bit under
  [`set.seed()`](https://rdrr.io/r/base/Random.html) and handles
  rank-deficient kernels (e.g. squared-exp with zero nugget) correctly.
  `tf_smooth.tfd(method = "savgol")` now inlines a small Savitzky–Golay
  filter in place of the
  [`pracma::savgol()`](https://rdrr.io/pkg/pracma/man/savgol.html)
  dependency.

### Bug fixes

- [`tf_evaluate()`](https://tidyfun.github.io/tf/dev/reference/tf_evaluate.md)
  no longer returns values at the wrong positions when the requested arg
  contains duplicates
  ([\#236](https://github.com/tidyfun/tf/issues/236)).
- `tf_integrate(f, definite = FALSE)` for irregular `tfd` with n \> 1 no
  longer crashes; the antiderivative’s per-curve grids are kept as a
  list ([\#237](https://github.com/tidyfun/tf/issues/237)).
- [`Math.tfd()`](https://tidyfun.github.io/tf/dev/reference/tfgroupgenerics.md)
  /
  [`Math.tfb()`](https://tidyfun.github.io/tf/dev/reference/tfgroupgenerics.md)
  now forward `...` to the underlying op so `round(x, digits)`,
  `log(x, base)`, `signif(x, digits)`, etc. honor their extra arguments
  instead of silently dropping them
  ([\#246](https://github.com/tidyfun/tf/issues/246)).
- [`tf_integrate()`](https://tidyfun.github.io/tf/dev/reference/tf_integrate.md)
  on irregular `tfd` no longer silently returns `NA` under default
  limits; for irregular input the defaults are now each curve’s own
  observed arg range (i.e., the range of its
  [`tf_arg()`](https://tidyfun.github.io/tf/dev/reference/tfmethods.md)
  values). Pass explicit `lower` / `upper` (or an extrapolating
  evaluator) to override
  ([\#253](https://github.com/tidyfun/tf/issues/253)).

### Vector-valued functional data

This release introduces first-class support for vector-valued
(multivariate) functional data – functions whose codomain is `R^d` –
alongside the existing univariate `tfd`/`tfb` classes.

- [`tfd_mv()`](https://tidyfun.github.io/tf/dev/reference/tfd_mv.md) and
  [`tfb_mv()`](https://tidyfun.github.io/tf/dev/reference/tfb_mv.md):
  new `vctrs`-based S3 classes for vector-valued functional data,
  holding several component functions per observation on a shared
  domain. Constructors accept named lists of `tfd`/`tfb` vectors or
  list-columns of matrices.
- [`tfb_mfpc()`](https://tidyfun.github.io/tf/dev/reference/tfb_mfpc.md)
  implements multivariate functional principal component analysis (Happ
  & Greven, 2018) for `tf_mv` data: a single set of scalar scores per
  curve shared across all components, with vector-valued eigenfunctions.
  Component weighting is configurable (`"inverse_variance"` default,
  `"snr"`, `"equal"`, or user-supplied). New data can be projected onto
  a fitted basis via
  [`tf_rebase()`](https://tidyfun.github.io/tf/dev/reference/tf_rebase.md)
  / `vec_cast()`. Accessors
  [`tf_mfpc_scores()`](https://tidyfun.github.io/tf/dev/reference/tfb_mfpc.md),
  [`tf_mfpc_efunctions()`](https://tidyfun.github.io/tf/dev/reference/tfb_mfpc.md)
  and the predicate
  [`is_tfb_mfpc()`](https://tidyfun.github.io/tf/dev/reference/tfb_mfpc.md).
- Multivariate registration:
  [`tf_register()`](https://tidyfun.github.io/tf/dev/reference/tf_register.md)
  gains `method = "srvf_mv"` for jointly aligning the components of
  `tf_mv` curves via the multivariate SRVF framework, and
  [`tf_register_shape()`](https://tidyfun.github.io/tf/dev/reference/tf_register_shape.md)
  provides elastic shape registration
  (rotation/translation/scale-invariant) via `fdasrvf`.
- New geometry verbs for `tf_mv` (and where meaningful for univariate
  `tf`):
  [`tf_norm()`](https://tidyfun.github.io/tf/dev/reference/tf_geom.md),
  [`tf_inner()`](https://tidyfun.github.io/tf/dev/reference/tf_geom.md),
  [`tf_tangent()`](https://tidyfun.github.io/tf/dev/reference/tf_geom.md),
  [`tf_arclength()`](https://tidyfun.github.io/tf/dev/reference/tf_arclength.md).
- Component accessors
  ([`tf_ncomp()`](https://tidyfun.github.io/tf/dev/reference/tf_mv_methods.md),
  [`tf_components()`](https://tidyfun.github.io/tf/dev/reference/tf_mv_methods.md),
  [`tf_component()`](https://tidyfun.github.io/tf/dev/reference/tf_mv_methods.md)),
  [`tf_split()`](https://tidyfun.github.io/tf/dev/reference/tf_splitcombine.md)
  /
  [`tf_combine()`](https://tidyfun.github.io/tf/dev/reference/tf_splitcombine.md)
  extensions and `[`/`[[` methods for extracting, replacing and
  recombining components.
- Function-wise scalar summaries
  [`tf_fmean()`](https://tidyfun.github.io/tf/dev/reference/functionwise.md),
  [`tf_fvar()`](https://tidyfun.github.io/tf/dev/reference/functionwise.md)
  and
  [`tf_fsd()`](https://tidyfun.github.io/tf/dev/reference/functionwise.md)
  return component-wise matrices for `tf_mv` objects.
- [`tf_interpolate()`](https://tidyfun.github.io/tf/dev/reference/tf_interpolate.md),
  [`tf_fwise()`](https://tidyfun.github.io/tf/dev/reference/functionwise.md),
  [`tf_sparsify()`](https://tidyfun.github.io/tf/dev/reference/tf_jiggle.md)
  and
  [`tf_jiggle()`](https://tidyfun.github.io/tf/dev/reference/tf_jiggle.md)
  now have component-wise `tf_mv` methods;
  [`tf_sparsify()`](https://tidyfun.github.io/tf/dev/reference/tf_jiggle.md)
  and
  [`tf_jiggle()`](https://tidyfun.github.io/tf/dev/reference/tf_jiggle.md)
  keep shared component grids by default via `same_arg = TRUE`.
- [`tf_where()`](https://tidyfun.github.io/tf/dev/reference/tf_where.md)
  /
  [`tf_anywhere()`](https://tidyfun.github.io/tf/dev/reference/tf_where.md)
  work on `tf_mv` objects with *joint* conditions across components,
  referenced by name (e.g. `tf_where(f, x > 0 & y < 1)`); there is no
  `value` column for `tf_mv` input. Components must share a common grid,
  or `arg` must be supplied explicitly.
- [`quantile()`](https://rdrr.io/r/stats/quantile.html) works on `tf_mv`
  objects, returning the *component-wise* pointwise quantiles: a `tf_mv`
  with one curve per requested probability in `probs`, exactly as
  `quantile.tf()` does per component (`na.rm` / `probs` / `type` are
  forwarded).
- [`points()`](https://rdrr.io/r/graphics/points.html) works on `tf_mv`
  objects, mirroring
  [`lines.tf_mv()`](https://tidyfun.github.io/tf/dev/reference/plot.tf_mv.md):
  in `"trajectory"` mode (`d == 2`) it overlays the paired
  `(x(t), y(t))` points with per-curve graphical-parameter recycling; in
  `"facet"` mode it overlays each component onto the current device.
- [`tf_depth()`](https://tidyfun.github.io/tf/dev/reference/tf_depth.md)
  gains a `tf_mv` method: a weighted componentwise aggregation of the
  (centrality) univariate depths, with a `weights` argument mirroring
  [`tfb_mfpc()`](https://tidyfun.github.io/tf/dev/reference/tfb_mfpc.md)
  (`"equal"` default, `"inverse_variance"`, or a numeric vector). The
  up-down ordering index `"MHI"` has no canonical vector-valued analogue
  and aborts with a pointer to
  [`tf_order()`](https://tidyfun.github.io/tf/dev/reference/tf_order.md)
  ([\#273](https://github.com/tidyfun/tf/issues/273)).
- [`median()`](https://rdrr.io/r/stats/median.html),
  [`summary()`](https://rdrr.io/r/base/summary.html) and
  [`fivenum()`](https://tidyfun.github.io/tf/dev/reference/fivenum.md)
  on `tf_mv` now select *observed* vector-valued curves via a single
  joint-depth index (one curve across all components), fixing the
  previous component-wise “chimera” where each component could come from
  a different observation. Pointwise entries (mean, min/max, the
  central-half range) remain component-wise
  ([\#273](https://github.com/tidyfun/tf/issues/273)).
- New generic
  [`tf_order()`](https://tidyfun.github.io/tf/dev/reference/tf_order.md)
  with a `tf_mv` method: `tf_order(f, by = "norm")` (or
  `by = <component name>`) orders vector-valued curves by an explicit
  scalar reduction. [`sort()`](https://rdrr.io/r/base/sort.html),
  [`rank()`](https://tidyfun.github.io/tf/dev/reference/tf_order.md) and
  [`xtfrm()`](https://rdrr.io/r/base/xtfrm.html) remain permanently
  undefined for `tf_mv` – there is no canonical total order on `R^d` –
  so base
  [`sort()`](https://rdrr.io/r/base/sort.html)/[`order()`](https://rdrr.io/r/base/order.html)/[`rank()`](https://tidyfun.github.io/tf/dev/reference/tf_order.md)
  fail fast rather than fabricate an order
  ([\#273](https://github.com/tidyfun/tf/issues/273)).

#### Contract change

- [`is_tf()`](https://tidyfun.github.io/tf/dev/reference/tfmethods.md)
  now returns `TRUE` for `tf_mv` as well as univariate `tfd`/`tfb`. Code
  that branched on
  [`is_tf()`](https://tidyfun.github.io/tf/dev/reference/tfmethods.md)
  to mean “univariate `tf`” should switch to the new predicate
  [`is_tf_1d()`](https://tidyfun.github.io/tf/dev/reference/tfmethods.md).
- `"arg"` and `"id"` are reserved and cannot be used as component names
  – they collide with the grid and curve-id columns of long-format
  conversions.
- Constructing a `tf_mv` from components with differing domains now
  *warns* when widening `tfd` components to the union domain
  (evaluations outside the observed range are `NA`) and *aborts* for
  `tfb` components, where basis evaluation outside the fitted range
  would extrapolate, i.e. fabricate values.
- Component-wise operations on a `tfb_mfpc`
  ([`tf_smooth()`](https://tidyfun.github.io/tf/dev/reference/tf_smooth.md),
  [`tf_derive()`](https://tidyfun.github.io/tf/dev/reference/tf_derive.md),
  [`tf_zoom()`](https://tidyfun.github.io/tf/dev/reference/tf_zoom.md),
  [`tf_integrate()`](https://tidyfun.github.io/tf/dev/reference/tf_integrate.md),
  [`mean()`](https://rdrr.io/r/base/mean.html)/[`sd()`](https://tidyfun.github.io/tf/dev/reference/tfsummaries.md)/[`var()`](https://tidyfun.github.io/tf/dev/reference/tfsummaries.md),
  `Summary` group generics) now consistently demote to valid
  per-component `tfb_fpc` representations with a one-time warning, like
  arithmetic already did – previously they silently produced objects
  that abort on later use.
- [`tf_fmax()`](https://tidyfun.github.io/tf/dev/reference/functionwise.md),
  [`tf_fmin()`](https://tidyfun.github.io/tf/dev/reference/functionwise.md)
  and
  [`tf_fmedian()`](https://tidyfun.github.io/tf/dev/reference/functionwise.md)
  on `tf_mv` return an `n x d` curves-x-components matrix like
  [`tf_fmean()`](https://tidyfun.github.io/tf/dev/reference/functionwise.md)/[`tf_fsd()`](https://tidyfun.github.io/tf/dev/reference/functionwise.md)
  (previously: a misnamed interleaved vector).
- [`tfb_fpc()`](https://tidyfun.github.io/tf/dev/reference/tfb_fpc.md)
  and
  [`tfb_mfpc()`](https://tidyfun.github.io/tf/dev/reference/tfb_mfpc.md)
  abort informatively when the input contains completely missing curves
  (partially missing evaluations continue to be handled by the
  soft-impute SVD); re-scoring new data with `NA` curves via
  [`tf_rebase()`](https://tidyfun.github.io/tf/dev/reference/tf_rebase.md)
  / `vec_cast()` yields `NA` scores and `NA` entries.

#### Tabular export of components on differing grids

- `as.data.frame.tf_mv(unnest = TRUE)` gains a `grids` argument
  controlling where components are evaluated when they live on different
  argument grids: `"union"` (default, previous behavior) evaluates every
  component on each curve’s union grid, interpolating inside the
  observed range; `"component"` evaluates each component strictly on its
  own grid (or on `arg`), so no values are fabricated where a component
  was not observed – the faithful export used by `tidyfun::tf_unnest()`.
  For shared grids both agree.

### More bug fixes (pre-release review)

- Empty prototypes
  ([`tfd()`](https://tidyfun.github.io/tf/dev/reference/tfd.md),
  [`tfb()`](https://tidyfun.github.io/tf/dev/reference/tfb.md),
  `tfd_mv(list())` and friends) carry the sentinel domain `c(NA, NA)`
  and now combine cleanly with populated vectors in all `vctrs` paths
  (`vec_c()`, `vec_rbind()`, [`c()`](https://rdrr.io/r/base/c.html),
  casts) – binding rows onto an initially empty `tf` column works again.
- [`tfb_mv()`](https://tidyfun.github.io/tf/dev/reference/tfb_mv.md) on
  an existing `tfb_mv` with an explicit `basis` argument re-fits in the
  requested basis instead of silently returning the old one; changing
  the basis kind converts through
  [`tfd()`](https://tidyfun.github.io/tf/dev/reference/tfd.md).
- [`tfb_mv()`](https://tidyfun.github.io/tf/dev/reference/tfb_mv.md)/[`tfb_mfpc()`](https://tidyfun.github.io/tf/dev/reference/tfb_mfpc.md)
  on raw lists/arrays route constructor arguments (`arg`, `domain`) and
  basis arguments to the right place instead of forwarding everything
  everywhere.
- [`tfd_mv()`](https://tidyfun.github.io/tf/dev/reference/tfd_mv.md) on
  3-d arrays keeps length-1 curve/arg margins and curve names.
- [`tf_derive()`](https://tidyfun.github.io/tf/dev/reference/tf_derive.md)
  and
  [`tf_integrate()`](https://tidyfun.github.io/tf/dev/reference/tf_integrate.md)
  on `tf_mv` accept per-component `arg` lists, consistent with
  [`tf_interpolate()`](https://tidyfun.github.io/tf/dev/reference/tf_interpolate.md).
- `Summary` group generics on `tf_mv` no longer recycle `NA` masks
  across operands of different lengths.
- [`tf_arclength()`](https://tidyfun.github.io/tf/dev/reference/tf_arclength.md):
  `method = "polyline"` honors `lower`/`upper` without creating
  near-duplicate vertices, and `method = "derive"` no longer NA-poisons
  irregular curves that do not span the full domain.
- Indefinite
  [`tf_integrate()`](https://tidyfun.github.io/tf/dev/reference/tf_integrate.md)
  on irregular `tfd` snaps user-supplied limits to (almost-)coinciding
  grid points instead of constructing invalid objects.
- [`tf_register()`](https://tidyfun.github.io/tf/dev/reference/tf_register.md)
  works for `tf_mv` inputs whose components live on different grids;
  [`tf_register_shape()`](https://tidyfun.github.io/tf/dev/reference/tf_register_shape.md)
  returns a user-supplied `template` unchanged from
  [`tf_template()`](https://tidyfun.github.io/tf/dev/reference/tf_registration.md);
  single-component `srvf_mv` delegates to univariate elastic
  registration and template-free single-curve registration is rejected
  with an explanation instead of crashing inside `fdasrvf`.

## tf 0.4.0

CRAN release: 2026-03-17

### New features

- [`tf_register()`](https://tidyfun.github.io/tf/dev/reference/tf_register.md)
  & co implement registration methods (SRVF, affine, landmark,
  continuous-criterion) for aligning functions and analyzing phase
  variability. Returns a new `tf_registration` class with accessors
  [`tf_aligned()`](https://tidyfun.github.io/tf/dev/reference/tf_registration.md),
  [`tf_inv_warps()`](https://tidyfun.github.io/tf/dev/reference/tf_registration.md),
  [`tf_template()`](https://tidyfun.github.io/tf/dev/reference/tf_registration.md).
- [`tf_depth()`](https://tidyfun.github.io/tf/dev/reference/tf_depth.md)
  now implements five functional depth methods: MBD (default), MHI, FM
  (Fraiman-Muniz), FSD (Functional Spatial Depth), and RPD (Regularized
  Projection Depth). MHI ranks functions from lowest (0) to highest (1);
  all other depths are scaled from 0 (most extreme) to 1 (most central).
  MBD is now scaled to \[0, 1\] instead of \[0, 0.5\].
- [`tf_split()`](https://tidyfun.github.io/tf/dev/reference/tf_splitcombine.md)
  /
  [`tf_combine()`](https://tidyfun.github.io/tf/dev/reference/tf_splitcombine.md)
  for splitting functions into sub-domain fragments and re-combining
  them
- [`tf_rgp()`](https://tidyfun.github.io/tf/dev/reference/tf_rgp.md)
  gains a `"brownian_bridge"` covariance option and can generate
  irregular data via the new `irreg` argument
- [`tfb_spline()`](https://tidyfun.github.io/tf/dev/reference/tfb_spline.md)
  now converts [`fda::fd`](https://rdrr.io/pkg/fda/man/fd.html) and
  `fda::fdSmooth` objects directly
- `mgcv`-style Fourier basis constructor for use in
  [`tfb_spline()`](https://tidyfun.github.io/tf/dev/reference/tfb_spline.md)
- sparklines in [`print()`](https://rdrr.io/r/base/print.html) /
  [`format()`](https://rdrr.io/r/base/format.html) for all `tf` subtypes
- [`tf_where()`](https://tidyfun.github.io/tf/dev/reference/tf_where.md)
  now defaults to `arg = tf_arg(x)`, consistent with other functions
- [`as.matrix()`](https://rdrr.io/r/base/matrix.html) for irregular
  `tfd` now always interpolates to a common grid

### Bug fixes

- [`tf_derive()`](https://tidyfun.github.io/tf/dev/reference/tf_derive.md)
  for `tfd` objects now uses second-order accurate finite differences,
  preserving the original domain and grid
  ([\#59](https://github.com/tidyfun/tf/issues/59))
- major overhaul of NA handling: NA entries are now consistently
  represented as `NULL` internally; fixes for
  [`tf_arg()`](https://tidyfun.github.io/tf/dev/reference/tfmethods.md),
  [`tf_evaluate()`](https://tidyfun.github.io/tf/dev/reference/tf_evaluate.md),
  [`tf_smooth()`](https://tidyfun.github.io/tf/dev/reference/tf_smooth.md),
  [`tfd()`](https://tidyfun.github.io/tf/dev/reference/tfd.md)
  conversion, [`is.na()`](https://rdrr.io/r/base/NA.html), printing, and
  arithmetic with NA entries
- fix
  [`summary.tf()`](https://tidyfun.github.io/tf/dev/reference/tfsummaries.md)
  returning wrong central range
- fix `tfd` domain enforcement for length-1 data
  ([\#139](https://github.com/tidyfun/tf/issues/139))
- fix printing for empty `tfb` objects and all-NA vectors

### Other changes

- `tf_derive` and `tf_integrate` can now handle `tfb` with non-identity
  link functions by falling back to `tfd`-calculus methods (and
  returning `tfd` objects).
- `tfd_irreg` arithmetic now operates on intersection of arg-values
  instead of failing when arg-values are not identical
- `tfb` arithmetic fix
- dropped `glue` dependency in favour of `cli` / `sprintf`; replaced
  [`stopifnot()`](https://rdrr.io/r/base/stopifnot.html) with
  informative
  [`cli::cli_abort()`](https://cli.r-lib.org/reference/cli_abort.html)
  throughout
- use
  [`vctrs::vec_arith()`](https://vctrs.r-lib.org/reference/vec_arith.html)
  for group generics, `vctrs` utils in `[.tf`
- various internal performance improvements
- use `air` formatter for all R code
